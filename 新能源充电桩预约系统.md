# 产品需求文档（PRD）：新能源充电桩智能预约平台

## 1. 产品背景

**市场需求与痛点**：
1.  **充电资源错配与效率低下**：截至2023年底，中国新能源汽车保有量已突破2000万辆，公共充电桩保有量约860万台，车桩比约为2.3:1。然而，充电桩平均利用率不足15%。核心矛盾并非总量不足，而是**时空错配**。用户痛点集中在：高峰期（如晚6-10点）热门站点“一桩难求”，而周边站点却有空闲；到达现场后发现充电桩故障或被燃油车占位，白跑一趟。
2.  **现有解决方案的缺陷**：当前主流充电APP（如特来电、星星充电）主要提供“查找-导航-即到即充”模式。该模式存在三大问题：
    *   **信息不透明**：仅显示“空闲/占用”状态，无法预知未来时段占用情况，用户决策依赖运气。
    *   **无资源锁定机制**：用户无法提前锁定充电桩，行程存在不确定性，尤其在长途出行或时间紧迫时焦虑感强。
    *   **运营商管理粗放**：无法通过预约机制平抑波峰波谷，导致设备利用不均衡，高峰期拥堵，闲时资源浪费。

**我们要解决的问题**：
构建一个以**预约机制**为核心的智能充电服务平台，解决新能源车主充电的确定性问题，并帮助运营商提升充电桩利用效率和运营收益。核心是**将充电从“随机事件”变为“计划事件”**。

## 2. 产品目标

**业务目标（可量化）**：
1.  **提升运营商核心收益**：上线6个月内，接入平台的充电桩整体**平均利用率提升至25%以上**（基线15%）。
2.  **优化用户充电体验**：将车主从出发到成功开始充电的**平均决策与等待时间缩短40%**（从平均25分钟缩短至15分钟内）。
3.  **提高平台粘性**：实现**预约订单占比达到总订单量的30%**。
4.  **扩大网络效应**：上线首年，接入平台的优质充电场站（利用率>20%）数量达到1000个。

**用户目标**：
1.  **对车主**：提供确定性的充电服务，让车主能够提前规划行程，减少无效等待和焦虑。
2.  **对运营商**：提供精细化的运营工具，通过价格杠杆和预约调度，提升单桩营收和整体运营效率。

**产品定位**：
一款连接新能源车主与充电运营商的**充电桩预约与智能调度平台**，通过“可预约、可锁定、可规划”的服务，让充电更高效、更确定。

## 3. 用户画像

**1. 新能源车主（核心用户）**
*   **角色**：私家车车主，通勤距离较长或家庭无固定车位充电桩。
*   **核心场景**：工作日下班后前往商场/公共充电站充电；周末长途出行前规划途中补电。
*   **使用频率/习惯**：每周充电1-3次，习惯在出发前或行程中使用手机APP寻找充电桩，对价格敏感，但更看重时间和确定性。
*   **痛点**：找不到可用桩、找到桩但需排队、到达后发现桩故障或油车占位、充电费用不透明。

**2. 充电桩运营商（客户与合作伙伴）**
*   **角色**：充电场站运营企业或拥有充电桩资产的物业公司。
*   **核心场景**：日常监控各站点桩群状态与收益；设置不同时段价格以调节流量；处理用户上报的桩体故障。
*   **使用频率/习惯**：每日登录后台查看数据，每周进行运营分析，不定期调整价格策略。
*   **痛点**：高峰期过载、闲时空置，设备利用率低；用户投诉多（占位、故障），处理成本高；缺乏数据驱动精细化运营的能力。

**3. 系统管理员（内部用户）**
*   **角色**：平台技术运营人员。
*   **核心场景**：审核新接入的运营商资质；配置全局系统参数（如佣金比例、通知模板）；监控系统服务健康度。
*   **使用频率/习惯**：按需操作，但需7x24小时响应系统告警。
*   **痛点**：确保系统高可用性、数据安全与业务流程稳定。

## 4. 功能需求

### P0 核心功能（MVP必须上线）

**F1：充电桩查询与预约**
*   **做什么**：车主基于地图，查询可预约的充电桩，并完成预约。
*   **怎么做**：
    1.  **智能展示**：首页地图默认展示用户周边充电站，以不同图标区分“即时可用”和“仅可预约”。列表视图支持按距离、价格、功率筛选。
    2.  **精准预约**：点击具体充电桩，展示未来24小时内的可预约时段块（如以30分钟为粒度）。灰色为不可用/已被预约，绿色为可选。
    3.  **预约锁定**：用户选择时段并确认后，系统为该用户锁定该桩位，同时向运营商系统发送锁定指令。生成预约订单，状态为“待使用”。
*   **验收标准**：
    *   地图加载时间 < 2秒。
    *   从进入详情页到成功预约，核心路径点击次数 ≤ 4次。
    *   同一时段同一充电桩，仅能被一个有效订单锁定。

**F2：充电状态实时查看与控制**
*   **做什么**：车主在充电过程中，实时监控进度，并具备远程停止充电的能力。
*   **怎么做**：
    1.  **状态同步**：车主通过APP或扫码启动充电后，进入充电监控页。该页面通过WebSocket与后端及运营商系统保持长连接，实时更新充电电量（kWh）、功率（kW）、已充电时长、预估剩余时间、当前累计费用。
    2.  **异常提醒**：如充电异常中断（如桩故障、网络断开），页面顶部显示强提醒，并引导用户操作（如重新连接或联系客服）。
    3.  **远程停止**：提供“停止充电”按钮，用户确认后可远程下发停止指令至充电桩。
*   **验收标准**：
    *   充电数据刷新延迟 < 3秒。
    *   从点击“停止充电”到桩端实际停止动作，延迟 < 10秒。
    *   异常状态提醒推送到达率 > 99.9%。

**F3：在线支付与自动结算**
*   **做什么**：充电结束后，系统自动生成账单，支持车主便捷支付，并完成与运营商的分账。
*   **怎么做**：
    1.  **无感支付**：充电结束后，自动跳转至支付页面，账单清晰列明服务费、电费、预约费（如有）。
    2.  **多方式支付**：默认使用车主设置的“首选支付方式”（如微信支付、支付宝、平台钱包）进行免密支付。用户也可切换其他方式。
    3.  **自动分账**：支付成功后，系统根据与运营商约定的分账规则（如平台收取X%服务费），自动将款项结算至运营商虚拟账户。
*   **验收标准**：
    *   支付成功率达到行业平均水平（>99.5%）。
    *   从充电结束到账单生成时间 < 2秒。
    *   分账计算准确率100%。

### P1 重要功能（提升体验与运营效率）

**F4：车主个人中心**
*   **做什么**：管理车主账户信息、历史记录及偏好设置。
*   **验收标准**：支持查看近一年的充电订单详情（含电量、费用、地点）。

**F5：运营商管理后台**
*   **做什么**：为运营商提供充电桩、订单、财务的数据看板与管理工具。
*   **验收标准**：运营商可自主上架/下架充电桩、设置基于时段的浮动价格（至少支持24小时粒度）。

### P2 优化功能（增强粘性与自动化）

**F6：智能提醒与通知系统**
*   **做什么**：在关键节点通过APP Push和短信通知用户。
*   **验收标准**：预约开始前15分钟，Push送达率 > 95%。

### 非功能需求
*   **性能**：核心页面（地图、充电监控）接口P95响应时间 < 800ms。系统能支持单日10万笔订单的生成与处理。
*   **安全**：用户支付信息加密存储（符合PCI DSS标准）。通信全程使用HTTPS。防刷单、防恶意占桩机制。
*   **可用性**：系统整体可用性SLA ≥ 99.9%。充电桩状态信息与运营商系统同步延迟 < 1分钟。

## 5. 用户流程说明

**核心业务流程1：车主预约充电**
1.  **启动查询**：车主打开APP，系统获取位置，展示地图与充电站列表。
2.  **筛选与决策**：车主使用筛选器（如快充、低价）或直接点击地图图标，查看站点详情。
3.  **选择预约**：在充电桩详情页，浏览未来时段可用性，选择一个绿色时段块。
4.  **确认与锁定**：确认预约时段和价格，点击“预约”。系统校验并发后，锁定资源，生成预约订单。
    *   **关键决策点**：时段选择、价格确认。
    *   **关键节点**：资源锁定成功。

**核心业务流程2：车主充电与支付**
1.  **启动充电**：车主在预约时段内到达现场，在APP订单页点击“开始充电”或扫描桩上二维码。系统验证预约有效性后，向桩下发启动指令。
2.  **实时监控**：进入充电监控页，车主可查看实时数据，也可随时手动停止。
3.  **结束与账单**：充电完成（自动或手动停止），系统从运营商处获取最终充电数据，生成账单。
4.  **支付**：跳转支付页，车主确认金额并完成支付。支付成功后订单状态更新为“已完成”。
    *   **关键决策点**：是否手动停止、选择支付方式。
    *   **关键节点**：充电启动成功、支付成功。

**核心业务流程3：运营商管理桩与订单**
1.  **登录查看**：运营商登录后台，首页查看核心仪表盘（今日订单量、营收、桩群状态分布）。
2.  **桩管理**：进入“充电桩管理”列表，可对单个桩进行“维护/上线”状态切换，或修改其价格策略。
3.  **订单管理**：进入“订单管理”，可筛选查看所有历史订单，处理异常订单争议。
    *   **关键决策点**：设置价格策略、处理异常订单。
    *   **关键节点**：价格策略生效、桩状态变更同步至前端。

## 6. 页面说明

**车主端APP核心页面清单**：
1.  **首页/发现页**：核心元素-地图组件、筛选栏、充电站列表。交互-点击地图图标或列表项，跳转至充电站详情页。
2.  **充电站/桩详情页**：核心元素-站点基本信息、桩列表、可预约时段选择日历、价格明细。交互-选择时段后，底部浮现“预约”按钮。
3.  **预约确认页**：核心元素-预约信息汇总（桩位置、时段、价格）。交互-点击“确认支付”跳转支付流程。
4.  **充电监控页**：核心元素-实时数据仪表、充电曲线图、停止充电按钮。交互-数据自动刷新，按钮触发远程指令。
5.  **订单详情/支付页**：核心元素-订单状态、充电详情、支付金额、支付方式选择器。交互-选择支付方式后调用第三方支付SDK。
6.  **我的（个人中心）**：核心元素-我的车辆、订单列表、设置入口。

**页面核心跳转关系**：
`首页` -> `详情页` -> `预约确认页` -> (支付流程) -> `订单列表`
`订单列表`中“进行中”订单 -> `充电监控页`
`充电监控页`充电结束后 -> `订单详情/支付页`

## 7. 埋点需求

为验证产品目标，需监控以下核心指标：

**业务健康度指标**：
*   `total_orders`：总订单数（日/周/月）
*   `reservation_order_ratio`：预约订单数 / 总订单数
*   `avg_utilization_rate`：接入桩的平均利用率（总充电时长 / 总可用时长）
*   `gmv`：平台总交易流水

**用户体验指标**：
*   `search_to_reserve_conversion_rate`：从搜索页面到成功预约的转化率
*   `avg_decision_time`：从打开APP到成功预约的平均时长
*   `no_show_rate`：预约后未在约定时段内启动充电的订单比例
*   `payment_success_rate`：支付成功率

**功能使用指标**：
*   `map_usage_rate`：使用地图视图进行搜索的用户占比
*   `filter_usage`：各筛选条件的使用次数
*   `remote_stop_usage`：远程停止功能的使用次数与场景

**异常监控指标**：
*   `charge_start_failure_rate`：启动充电失败率
*   `data_sync_latency`：充电状态数据同步延迟分布
